## 1. 前置依赖

OpenClaw 需要以下工具：

| 依赖 | 版本要求 |
|------|---------|
| Node.js | 24（推荐），支持 22 LTS（22.16+） |
| Git | 任意版本 |

如果尚未安装，请先查看 [前置依赖安装指引](/signal-skills/prerequisites)。

## 2. 安装 OpenClaw

> 如果下载失败或超时，见下方[常见问题 — 网络代理配置](#网络代理配置)。

### 方式一：安装脚本（推荐）

使用官方安装脚本，自动处理 Node 检测、安装和新手引导：

**macOS / Linux / WSL2：**

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

**Windows (PowerShell)：**

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### 方式二：npm 安装

```bash
# 全局安装
npm install -g openclaw@latest

# 运行新手引导并安装守护进程
openclaw onboard --install-daemon
```

验证安装：

```bash
openclaw --version
```

---

## 3. 基础配置

安装脚本会自动运行新手引导，完成以下配置：
- 创建配置文件和状态目录（默认 `~/.openclaw/`）
- 安装并启动 Gateway 网关服务
- 引导配置 AI 模型 API Key
- 设置安全认证

如果跳过了新手引导，可以手动运行：

```bash
openclaw onboard --install-daemon
```

### 配置 API Key

API Key 在 `openclaw onboard` 新手引导过程中会自动引导配置。如果需要后续修改，可以重新运行 onboard 或在 OpenClaw 的 Web 界面中调整。

### 安全配置要点

OpenClaw 默认配置 `bind: "loopback"`，即只允许本机访问（127.0.0.1），同时会自动生成 token 用于认证。**切勿将 bind 修改为非 loopback 模式**，否则会暴露到局域网甚至公网。

### 验证配置

```bash
# 检查配置和系统状态
openclaw doctor

# 打开浏览器 UI
openclaw dashboard
```

---

## 4. 启动 OpenClaw

Gateway 网关服务已自动启动，直接打开浏览器 UI：

```bash
openclaw dashboard
```

访问 `http://localhost:18789`，使用配置的密码登录。

---

## 5. 连接 Signal Skills

### 步骤 A：获取 API Key

前往 [授权申请](/signal-client/apply) 页面获取 Signal Skills API Key。详细流程见 [快速开始](/signal-skills/getting-started)。

### 步骤 B：配置 MCP 连接

编辑配置文件 `~/.openclaw/openclaw.json`，添加 `mcpServers` 字段：

```json
{
  "mcpServers": {
    "trade-signal": {
      "type": "http",
      "url": "<Feed 服务地址>/mcp",
      "headers": {
        "x-api-key": "<你的 Signal Skills API Key>"
      }
    }
  }
}
```

### 步骤 C：安装技能包

将 [下载](/signal-skills/download) 并解压后的技能包放入 Skills 目录：

```bash
# Skills 目录路径
~/.openclaw/workspace/skills/

# 放入后的结构示例
~/.openclaw/workspace/skills/
└── trade-signal-skills/
    └── SKILL.md
```

配置完成后重启网关服务使其生效：

```bash
openclaw gateway restart
```

## 常见问题

### 网络代理配置

**安装时使用淘宝镜像（国内用户推荐）：**

```bash
npm install -g openclaw@latest --registry=https://registry.npmmirror.com
```

**运行时配置代理（访问 AI API）：**

```bash
# macOS/Linux
export HTTP_PROXY="http://127.0.0.1:7890"
export HTTPS_PROXY="http://127.0.0.1:7890"

# Windows (PowerShell)
$Env:HTTP_PROXY="http://127.0.0.1:7890"
$Env:HTTPS_PROXY="http://127.0.0.1:7890"
```

> 端口号 `7890` 以你本地代理工具的实际端口为准（Clash 7890、V2Ray 10809）。

### 无法访问 Web 界面

1. 确认 OpenClaw 已启动：`openclaw status`
2. 确认访问地址为 `http://localhost:18789`
3. 检查防火墙是否阻止了本地访问

### 如何更新

```bash
npm install -g openclaw@latest
```

### 如何卸载

```bash
# 推荐方式
openclaw uninstall

# 或手动卸载
npm rm -g openclaw
rm -rf ~/.openclaw  # macOS/Linux
```
